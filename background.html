Polygenic indices

Chapter 2 Background

To construct PGS, but this section will give some background on the different PGS methods, and some general things we should consider when QC’ing and working with genotypes and summary statistics.

To construct a polygenic score we need lalala. Below we will discuss first the genotype data, then the summary statistics, and finally the polygenic score methods.

We saw yesterday what a PGS is and how to include it in analyses, but how do you construct them in the first place?

You’ve seen how to QC genotype data. You’ve seen how to construct a PGS as the sum of SNPs weighted by their GWAS effect size estimates.

Many methods with different levels of complexity:

  • Naïve/basic method – use all effect estimates from GWAS directly.
  • PT+Clumping (PRSICE) – select specific SNPs to include based on GWAS estimates based on P-value and prune markers based on LD (to try to adjust for LD we don’t adjust effect estimates, instead we throw SNPs out!).
  • LDPred – select specific SNPs and correct effect estimates for SNP correlations (LD) (we don’t throw markers out to correct for LD but instead adjust the effect sizes!)
  • LASSOSUM – select specific SNPs and also correct for SNP correlations (LD) (we don’t throw markers out to correct for LD but instead adjust the effect sizes).
  • The continuous shrinkage PGS method? Can’t remember the name, look up the paper.

We will consider questions such as:

  • How do I make the most predictive score?
  • Which method is the best?
  • What’s the best choice of parameter XYZ?
  • As with any prediction problem: Tweak parameters/model, evaluate your model on hold-out data, iterate. Pick the model that scores highest on your evaluation metric.

General recipe for a polygenic score:

  1. We want to construct a PGI for phenotype, say, \(y_i\), where \(i\) is individual.
  2. In a discovery sample \(\mathcal{D}\), we run a simple GWAS for \(y\), say: \[y_i = \alpha + \beta_{j} SNP_{ij} + \varepsilon_{ij}, \quad i \in \mathcal{D}, j = 1, \ldots, J\] to get summary statistics \({\{}\hat{\beta_j}, se(\hat{\beta_j}){\}}_{j=1}^J\). Recall that \(SNP_{ij}\) denotes an allele count!
  3. In a validation sample \(\mathcal{T}\), we pick a set of SNPs \(\mathcal{J}\), and calculate adjusted effect sizes \(\tilde{\beta}_j = f(\hat{\beta}_j), j \in \mathcal{J}\) using some method. In this process we may rely on LD information from a reference sample \(\mathcal{R}\).
  4. Finally, in the analysis sample \(\mathcal{A}\), for each individual \(i \in \mathcal{A}\) we construct the score \[PGI_i = \sum_{j \in \mathcal{S}} \tilde{\beta}_j SNP_{ij}, \quad i \in \mathcal{A}\]

2.3 Datasets

This means we rely on several datasets (or splits of the same dataset):

  • Discovery sample You only need this if you do your own GWAS, but even if you use a ready-made set of summary statistics, you must make sure that the sample they used does not overlap with your analysis sample!
  • Validation sample Call it a training/validation sample. Use it to evaluate your choices of hyperparameters etc.
  • Reference sample Sometimes necessary, used to calculate SNP correlations (LD structure). Can overlap with validation/discovery sample, but generally not with analysis sample.
  • Analysis sample The sample in which you want to construct a polygenic index for each individual.

Requirements:

  • We want the discovery sample to be large!
  • The reference/training sample can overlap.
  • The analysis sample should not overlap with discovery/reference/training samples!

2.4 Polygenic score methods

2.4.1 Naive method

Idea: Weigh all SNPs together using the effect estimates from a GWAS. Say we have a simple GWAS of \(J\) univariate regressions in the discovery sample \(\mathcal{D}\): \[y_i = \beta_0 + \beta_{j} \text{SNP}_{ij} + \varepsilon_{ij}, \quad i \in \mathcal{D}\] which gives us summary statistics \({\{}\hat{\beta_j}, se(\hat{\beta_j}){\}}_{j=1}^J\). We could construct the polygenic index as: \[\hat{PGI}_i = \sum_{j=1}^J \hat{\beta}_j \text{SNP}_{ij}, \quad i \in \mathcal{A}\] where we use the summary statistics from the discovery sample to weigh the SNPs for the individuals in the analysis sample \(\mathcal{A}\).

Issues:

  • \(\hat{\beta}_j\) can be noisy though, how can we improve? What about incorporating the standard error \(se(\hat{\beta_j})\) and selecting SNPs based on their signficance?
  • SNPs are correlated (LD, linkage disequilibrium), especially at shorter ranges, but the GWAS is a univariate regression, so we do not control for neighbouring SNPs that are correlated with SNP \(j\). Can we account for this somehow? Either in our selection of SNPs, or by adjusting the weights?

2.4.2 Pruning/clumping and threshold

Idea: Improve on the naive method by getting rid of SNPs that are highly correlated (in LD with eachother) and keep only SNPs that pass a test criterion.

Two approaches: Pruning and clumping.

  • Pruning: We start at snp \(j\), and test correlations at \(j+1, j+2, \ldots, j+M\) if the correlation exceeds a threshold then \(j\) is dropped.
  • Clumping: We start at snp \(j\), and test correlations at \(j+1, j+2, \ldots, j+M\), if the correlation exceeds a threshold then the SNP with the highest p-value is dropped.

Clumping/thresholding requires us to select the p-value threshold, the squared correlation threshold, and the window size. In PRSICE r2 is 0.1 with a window of 250kb. PRSICE only searches over p-val threshold, not the other two hyperparameters.

We can prune (i.e. remove) SNPs that are highly correlated by looking at correlation in a reference sample. We can implement select SNPs via hard thresholding using a p-value cut off. How do we select the threshold? Grid search. Let’s work through the recipe.

2.4.3 LDPred

Idea: Shrink effect estimates according to LD. Do bayesian selection on SNPs. Model: \[ yyyy \]

Two flavors: Infinitesimal (\(p = 1.0\)) and non-infinitesimal model (\(p \in (0; 1)\))..

  • The inf model reduces to a Gaussian prior on beta. Nice, this means it is fast as we can analytically derive the mean of the posterior:
  • The non-model is equivalent to a spike-slab model, so no analytical solution. We will have to sample the density. Slow!

How do we select \(p\)? Grid search, or a hyperprior and estimate it along with adjusted effect sizes. A lot of papers sticks to LDPred-Inf. To incorporate LD information we need the pairwise correlations between SNPs. We can call this correlation matrix the LD matrix, it is a triangular matrix, and, if we disregard long-range LD, a banded matrix. Nice! LDPred does not use LD blocks, but uses a sliding window around the current SNP when constructing this LD matrix.

LDPred1 is largely similar, but written in Python. LDPred2 is significantly faster and easier to work with from R, as it is not a commandline tool (if you can, then why not avoid platform dependent shell scripting).
